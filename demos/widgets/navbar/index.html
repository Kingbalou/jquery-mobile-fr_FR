Barre de navigation API

jQuery Mobile a un widget de barre de navigation très simple qui est utile pour fournir un maximum de 5 boutons avec des icônes en option dans une barre.

Barre de navigation basique

Une barre de navigation est codée comme une liste non ordonnée de liens enveloppée dans un élément de conteneur qui contient l'attribut data-role="navbar". Quand un lien dans la barre de bavigation est cliqué, il porte le statut actif (selected). La classe ui-btn-active est d'abord supprimée de toutes les ancres dans la barre de navigation avant d'être ajouté au lien activé. Si c'est un lien vers une autre page, la classe sera à nouveau supprimée après que la transition soit terminée.

Pour qu'un élément soit au statut actif, ajoutez class="ui-btn-active" à l'ancre correspondant dans votre balisage. Additionnellement, vous pouvez ajouter une classe ui-state-persist de façon que le framework restaure l'état actif à chaque fois que la page sera affichée tant qu'elle existe dans le DOM.

Les barres de navigation avec 1 élément sera rendu avec 100% de la largeur.

Barres de navigation dans les entêtes

Si vous souhaitez ajouter une barre de navigation en haut de la page, vous pouvez toujours avoir un titre de page et des boutons. Il suffit d'ajouter le conteneur de la barre de navigation à l'intérieur du bloc de l'entête, juste après le titre et les boutons dans l'ordre du code.

Persistante

Il y a également une variante, une barre de navigation persistante qui fonctionne davantage comme une barre d'onglets qui reste fixe pendant que vous naviguez à travers les pages. Pour qu'un élément soit au statut actif avant l'initialisation de la barre de navigation, ajoutez class="ui-btn-active" à l'ancre correspondant dans votre balisage. Additionnellement, vous pouvez ajouter une classe ui-state-persist de façon que le framework restaure l'état actif à chaque fois que la page sera affichée tant qu'elle existe dans le DOM.

Icônes

Les icônes peuvent être ajoutées aux éléments de la barre de navigation en ajoutant l'attribut data-icon en spécifiant une icône mobile standard pour chaque ancrage. Par défaut, les icônes sont ajoutées au-dessus du texte (data-iconpos="top"). Les exemples suivants ajoutent des icônes à la barre de navigation dans un pied de page.

Position d'icône

La position de l'icône est définie sur le conteneur de la barre de navigation plutôt que cela soit défini individuellement sur les liens par souci de cohérence visuelle. Par exemple, pour placer les icônes dessous les textes, ajoutez l'attribut data-iconpos="bottom" au conteneur de la la barre de navigation.

Thèmes

Les barres de navigation héritent du nuancier de thème de leur conteneur parent, tout comme les boutons. Si une la barre de navigation est placée dans l'entête ou le pied de page de la barre d'outils, elle héritera du nuancier "a" de la barre d'outils par défaut pour les barres, sauf si vous définissez ceci dans le balisage.

Voici quelques exemples de barres de navigation dans différents nuancier de conteneur qui automatiquement héritent de la lettre du nuancier de leurs parents. Notez que dans ces exemples, au lieu d'utiliser un attribut data-theme, nous ajoutons manuellement les classes de nuancier pour appliquer le nuancier du corps (ui-body-a) et la classe pour ajouter l'espacement standard du corps (ui-body), mais l'héritage fonctionne de la même manière :
