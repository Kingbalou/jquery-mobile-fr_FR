Grilles API

Le framework jQuery Mobile fournit un moyen simple de construire des colonnes basé sur du CSS qui peut être aussi responsive.

Grille basique

Les grilles sont de largeur 100%, totalement invisible (pas de bordure ou de fond d'écran) et n'ont pas d'espacement ou de marge, donc ils ne devraient pas interférer avec les styles des éléments placés à l'intérieur. Dans le conteneur de grille, les éléments enfants sont affectés ui-block-a/b/c/d/e d'une manière séquentielle qui fait que chaque élément de "bloc" se positionne côte à côte, formant la grille.

Grilles deux colonnes

Pour construire une mise en page à deux colonnes (50/50%), commencez avec un conteneur avec class de ui-grid-a. Et ajoutez deux conteneurs enfant à l'intérieur classé avec ui-block-a pour la première colonne et ui-block-b pour la seconde. Sur les blocs ci-dessous, nous ajoutons deux classes : ui-bar pour ajouter l'espacement de la barre par défaut et ui-bar-e pour appliquer le fond dégradé et le style de police pour le nuancier de thème "e" de la barre d'outils. A des fins d'illustration, un attribut d'alignement style="height:120px" est également ajouté à chaque grille pour mettre à chacun une hauteur standard.

Grilles à lignes multiples

Les grilles sont conçues pour envelopper de multiples lignes d'élément. Par exemple, si vous spécifiez une grille à 3 colonnes (ui-grid-b) sur un conteneur qui a neuf blocs enfant, il se terminera en 3 lignes de 3 éléments chacun. Il y a une règle CSS pour effacer les flotteurs et commencer une nouvelle ligne lorsque la class = ui-bloc-une est vu donc assurez-vous d'attribuer des classes de bloc dans une séquence répétée (a, b, c, a, b, c, etc) qui correspond au type de la grille :

Classe de grille seule

Le framework ajoute la marge à gauche et à droite des boutons dans une grille. Pour un unique bouton, vous pouvez utiliser un conteneur avec la classe ui-grid-solo et enveloppez le bouton dans un div avec la classe ui-block-a Comme pour l'exemple ci-dessous. De cette façon, le bouton recevra la même marge. Voir d'autres exemples de boutons dans les grilles.

Grille Responsive

C'est facile de prendre les grilles standard et les rendre responsive en empilant des blocs de grille à des largeurs étroites. Puisque nous voulons juste remplacer les positions et les largeurs des grilles standard sous un point d'arrêt unique, utilisez un point d'arrêt max-width pour appliquer le style empilé comme une substitution.

Nous recommandons l'ajout d'une classe (ex : my-breakpoint) pour étendre des styles pour la media query de sorte qu'ils puissent être appliqués de manière sélective. A partir de cette base, vous pouvez plus personnaliser l'apparence ou même ajouter des points d'arrêt supplémentaires. Voir un exemple de grille responsive personnalisée.


/* Empile toutes les grilles en dessous de 40em (640px) */
@media all and (max-width: 35em) {
	.my-breakpoint .ui-block-a,
	.my-breakpoint .ui-block-b,
	.my-breakpoint .ui-block-c,
	.my-breakpoint .ui-block-d,
	.my-breakpoint .ui-block-e {
		width: 100%;
		float: none;
	}
}

Point d'arrêt prédéfini

Pour appliquer un point d'arrêt prédéfini pour empiler les grilles en dessous de 35em (560px), ajouter la classe .ui-responsive au conteneur de la grille.
