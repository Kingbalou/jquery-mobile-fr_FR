Table : Reflow API

Le mode de table reflow fonctionne en pliant les colonnes de la table dans une présentation empilé qui ressemble à des blocs de label/données pour chaque ligne.

Reflow basiques

La table responsive reflow ne nécessite qu'une table avec un data-role = "table" sur l'élément table. Il n'est pas nécessaire de définir l'attribut data-mode car reflow est la valeur par défaut. N'oubliez pas d'inclure les éléments thead et tbody dans votre table. Cet exemple utilise également le point d'arrêt responsive prédéfini, appliqué via la classe ui-responsive.

Création de table responsive

Par défaut, une table avec le mode reflow affichera le style de présentation empilé sur toutes les largeurs d'écran. Les styles pour rendre la table responsive sont ajoutés en appliquant une media query avec des règles pour passer à la présentation de style tableau au-dessus d'une largeur d'écran spécifique.

Ceci est réalisé en enroulant quelques règles CSS simples et une media query qui s'applique uniquement aux règles en-dessus d'un point d'arrêt d'une certaine largeur. Les styles font apparaitre les lignes d'entête de table, affichent les cellules sous forme de tableau et masquent les éléments label générés au sein de chacune d'elle. Voici un exemple de media query qui change la présentation à 40em (640 pixels) : 

@media ( min-width: 40em ) {
	/* Affiche la ligne entête de la table et définit toutes les cellules à display: table-cell */
	.my-custom-breakpoint td,
	.my-custom-breakpoint th,
	.my-custom-breakpoint tbody th,
	.my-custom-breakpoint tbody td,
	.my-custom-breakpoint thead td,
	.my-custom-breakpoint thead th {
		display: table-cell;
		margin: 0;
	}
	/* Masque les labels dans chaque cellule */
	.my-custom-breakpoint td .ui-table-cell-label,
	.my-custom-breakpoint th .ui-table-cell-label {
		display: none;
	}
}

Il est préférable d'utiliser une classe sur la table pour appliquer le point d'arrêt. Ajoutez ces règles à votre feuille de style personnalisée qu'il faut inclure dans le head de la page. Nous vous recommandons de créer un ensemble de classes de point d'arrêt personnalisés qui peut être utilisé pour appliquer des points d'arrêt de table standard dans votre projet.

Choisir un point d'arrêt

L'objectif est de déterminer la largeur minimale à laquelle l' ensemble de la table s'adapte confortablement à l'intérieur de l'écran. Trouvez cette largeur en remplissant un tableau avec des données réalistes, puis réglez la fenêtre du navigateur jusqu'à ce que la table corresponde complètement et ait un peu d'espace supplémentaire pour tenir compte des différents rendus des appareils. C'est à cet endroit qu'il faut définir le point d'arrêt qui permute entre l'empilement et le mode de présentation en tableau. Le point d'arrêt de largeur dépend fortement du nombre de colonnes dans la table et le contenu de chaque cellule.

L'application d'un point d'arrêt prédéfini

Même si nous vous encourageons vivement à écrire vous-même des points d'arrêt personnalisé, le framework comprend un seul point d'arrêt pré-configuré qui cible le style empilé pour téléphones plus petits et le change pour une présentation en tableau sur les grands téléphones, les tablettes et les appareils de bureau. Pour utiliser ce point d'arrêt prédéfini, ajoutez la classe ui-responsive à la table pour convertir la présentation empilé pour une présentation sous forme de tableau à partir de 560px (35em). Si ce point d'arrêt ne fonctionne pas pour votre contenu, nous vous encourageons à écrire un point d'arrêt personnalisé tel que décrit ci-dessus.

<table data-role="table" class="ui-responsive">

Entêtes de colonnes groupées

Il est assez courant de devoir regrouper logiquement plusieurs colonnes sous une même entête de groupe pour les données financières ou scientifiques. Le framework peut supporter la version la plus simple de ceci, en acceptant deux rangées d'entêtes de table (TH), avec la première rangée contenant un simple attribut colspan pour grouper les colonnes en-dessous. Dans cette configuration, le framework ajoutera une classe à l'étiquette de la première cellule de chaque groupe pour vous permettre de styliser celles-ci différemment et fournir une hiérarchie visuelle supplémentaire.
