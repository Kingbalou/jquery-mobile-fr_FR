Cases à cocher API

Les cases à cocher sont utilisées pour fournir une liste d'options dont plusieurs peuvent être sélectionnés.

Balisage basique

Pour créer une seule case, ajoutez un input avec un attribut type="checkbox" et un label correspondant. Si le input n'est pas enveloppé dans le label correspondant, veuillez définir l'attribut for du label pour correspondre à l'id de l'input ainsi ils seront sémantiquement associés.

Taille mini

Pour avoir une version plus compacte, qui est utile dans les barres d'outils et les espaces restreints, ajoutez l'attribut data-mini="true" à l'élément pour créer une version mini.

Groupe vertical

Typiquement, il y a plusieurs cases à cocher figurant sous un titre de la question. Pour intégrer visuellement de multiples cases à cocher dans un ensemble groupé de bouton, le framework va automatiquement supprimer toutes les marges entre les boutons et arrondir seulement les coins supérieurs et inférieurs de l'ensemble s'il y a un attribut data-role="controlgroup" sur le fieldset.

Groupe horizontal

Les cases à cocher peuvent aussi être utilisé pour un ensemble de boutons groupés où plus d'un bouton peut être sélectionné à la fois, comme le groupe de boutons gras, italique et souligné vu dans un traitement de texte. Pour faire un jeu de bouton horizontale, ajoutez data-type="horizontal" à fieldset.

Horizontal :

Position d'icône

Pour changer la position de l'icône de sélection qui est par défaut sur ​​la gauche, ajoutez l'attribut data-iconpos="right" au fieldset pour le positionner à droite.

Icône à droite :

Thème

Pour définir le thème, ajoutez l'attribut data-theme sur le fieldset aux cases à cocher.
