Barres de navigation persistante API

Ces pages sont une démo d'une barre de navigation persistante dans une barre d'outils de pied de page fixe. Cliquez sur un des liens dans le pied de page, et vous pourrez voir la transtion du contenu de la page. L'entête et le pied de page persistants restent en place sur ces pages durant l'animation vers la nouvelle page HTML.

Pour que le framework applique le comportement persistant, ajouter le même attribut data-id aux entêtes et/ou aux pied de page de toutes les pages HTML dans la navigation. C'est simple : si la page où vous naviguez a une entête ou un pied de page avec les mêmes data-id, les barres d'outils apparaissent fixes en dehors de la transition. Chacune des pages dans cette démo a une transition différente pour tester comment cela fonctionne.

Généralement, la technique de la barre d'outils persistante sera combinée avec le positionnement fixe. Dans cet exemple, le pied de page a aussi une barre de navigation :


<div data-role="footer" data-id="foo1" data-position="fixed">
	<div data-role="navbar">
		<ul>
			<li><a href="a.html">Info</a></li>
			<li><a href="b.html">Amis</a></li>
			<li><a href="c.html">Albums</a></li>
			<li><a href="d.html">Emails</a></li>
		</ul>
	</div><!-- /navbar -->
</div><!-- /footer -->

Pour définir un statut actif sur l'un des éléments dans la barre d'outils persistante, ajouter une classe class="ui-btn-active ui-state-persist" correspondant à l'ancre.


<li><a href="a.html" class="ui-btn-active ui-state-persist">Info</a></li>

Une remarque à propos des transitions

Les transitions de page slide, slideup, slidedown, fade ou none fonctionnent correctement avec les barres d'outils persistantes. Toutefois, les transitions 3D intensives, comme flip, turn et flow, peuvent causer avec cette technique des problèmes de performance, de positionnement et d'animation, donc nous ne recommandons pas leur utilisation.
