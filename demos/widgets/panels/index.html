La position du panneau sur l'écran est définit par l'attribut data-position. La valeur par défaut de l'attribut est left, ce qui signifie qu'il apparaîtra à partir du bord gauche de l'écran. Spécifiez data-position="right" pour le faire apparaître à partir du bord droit.

Le mode d'affichage du panneau est définit par l'attribut data-display. La valeur de l'attribut par défaut est reveal, ce qui signifie que le panneau s'ouvrira à côté de la page et poussera la page plus loin.. Spécifiez data-display="overlay" pour le faire apparaitre au dessus du contenu de la page. Un troisième mode, data-display="push" anime à la fois le panneau et la page.

Où va le balisage du panneau dans le balisage

Un panneau doit être rattaché aux éléments header, content et footer à l'intérieur de la page jQuery Mobile. Vous pouvez ajouter le balisage du panneau soit avant ou après ces éléments, mais pas entre les deux. Un panneau ne peut pas être placé à l'extérieur d'une page, mais cette contrainte sera supprimée dans une future version.

Voici un exemple du panneau avant le header, content et footer dans l'ordre de la source :


<div data-role="page">

<div data-role="panel" id="mypanel">
    <!-- le contenu du panneau vient ici -->
</div><!-- /panel -->

<!-- header -->
<!-- content -->
<!-- footer -->

</div><!-- page -->

Alternativement, il est possible d'ajouter le balisage du panneau after le header, content et footer dans l'ordre de la source, juste avant la fin du conteneur de page. Selon l'endroit où vous placez le balisage du panneau, dépendra de la façon dont vous voulez que contenu de la page soit lu pour les personnes qui se trouvent sur la page dans un dispositif de grade C (HTML seulement) ou d'un lecteur d'écran.

Contenu dynamique

Lorsque vous ajoutez dynamiquement le contenu d'un panneau ou rendez visible un contenu caché lorsque le panneau est ouvert, vous devez déclencher l'événement updatelayout sur le panneau.


$( "#mypanel" ).trigger( "updatelayout" );

Le framework vérifiera ensuite la nouvelle hauteur du contenu du panneau. elle-ci dépasse la hauteur de l'écran, il réglera le min-height de la page à cette hauteur et détachera les panneaux avec data-position-fixed="true". Voir aussi le positionnement du panneau.

Ouverture d'un panneau

Pour contrôler un panneau depuis un lien, pointez le href à la référence du ID du panneau que vous voulez faire apparaitre (mypanel dans l'exemple ci-dessus). Ceci indique au framework de lier le lien au panneau. Ce lien permet de basculer l'affichage de la popup afin d'ouvrir le panneau en cliquant dessus et de le fermer en cliquant à nouveau dessus.

Fermeture d'un panneau

En cliquant sur ​​le lien qui a ouvert le panneau, en basculant à droite ou à gauche ou en tapant sur ​​la touche Echap, cela ferme le panneau. Pour désactiver le comportement de fermeture par le swipe, ajoutez l'attribut data-swipe-close="false" au panneau.

Par défaut, les panneaux peuvent être fermés en cliquant en dehors du panneau. Pour éviter cela, ajoutez l'attribut data-dismissible="false" au panneau. Il est possible d'avoir le panneau et la page côte à côte sur des largeurs d'écran plus larges et d'empêcher le comportement du clic en dehors du panneau qu'à partir d'une certaine largeur d'écran en appliquant une media query. Voir la section responsive ci-dessous pour plus de détails.

Il est fréquent d'ajouter un bouton de fermeture à l'intérieur du panneau. Pour ajouter le lien qui fermera le panneau, ajoutez l'attribut data-rel="close" pour dire au framework de le fermer quand il est cliqué. Il est important de s'assurer que ce lien est également disponible si JavaScript n'est pas disponible, c'est pourquoi nous recommandons que le href pointe sur l'ID de la page où l'utilisateur doit aller à la fermeture. Par exemple, si le bouton pour ouvrir le panneau est dans la barre d'entête qui a un ID my-header, le lien de fermeture dans le panneau doit être :


<a href="#my-header" data-rel="close">Fermer le panneau</a>

Animations du panneau

Les panneaux s'animeront si le navigateur prend en charge les transformations 3D, c'est les mêmes critères pour le support d'animation CSS que nous utilisons pour les transitions de page. Les animations du panneau utilisent la transformation CSS translate3d(x,y,z) pour s'assurer qu'ils sont fluides.

L'option animate permet de désactiver les animations du panneau pour tous les appareils. Pour désactiver les animations via le balisage, ajoutez l'attribut data-animate="false" au conteneur du panneau.

Positionnement du panneau

Le panneau sera affiché avec la propriété CSS position:absolute, cela permet de le faire défiler avec la page. Quand un panneau est ouvert, le framework vérifie si le bas du contenu du panneau est visible. Si ce n'est pas le cas, il défile vers le haut de la page.

Vous pouvez définir un panneau avec position:fixed, ainsi son contenu apparaîtra peu importe la distance au bas de la page que vous défiler, en ajoutant l'attribut data-position-fixed="true" au panneau. Le framework vérifie également si le contenu du panneau s'adapte à l'intérieur de la fenêtre avant d'appliquer le positionnement fixe car cette propriété empêche le contenu du panneau de défiler. overflow n'est pas bien supporté pour être utiliser à ce moment Si le contenu du panneau est trop long pour tenir dans la fenêtre, le framework affichera tout simplement le panneau sans positionnement fixe. Voir un exemple de panneaux avec position fixe.

Style des panneaux

Par défaut, les panneaux ont des styles très simples, ce qui vous permet de personnaliser selon les besoins. Les panneaux sont essentiellement de simples blocs, sans marge, qui se positionnent de chaque côté du contenu de la page. Le framework enveloppe le contenu du panneau dans un div avec la classe ui-pannel-inner qui a un espacement de 15 pixels. Si nécessaire, vous pouvez remplacer ceci avec un CSS personnalisé ou utiliser l'option classes.panelInner pour définir un nom de classe différent pour le div.

Les panneaux ont une largeur fixe de 17em (272 pixels), ce qui est assez étroit pour montrer encore une partie du contenu de la page, cela rend plus facile la possibilité de fermer en cliquant en dehors du panneau, et cela se présente bien sur un grand écran de tablette ou de bureau. Les styles pour définir la largeur des panneaux sont assez complexes, mais ceux-ci peuvent être remplacées avec des CSS selon les besoins.

A part, le thème de fond d'écran, la largeur et la hauteur à 100%, les panneaux sont peu stylisés. Le thème par défaut des panneaux est le "c". Vous pouvez définir un thème pour le panneau par l'ajout de l'attribut data-theme sur le conteneur du panneau ou définir data-theme="none" et ajouter selon les besoins vos propres classes.

Notez que l'ajout d'espacement, de bordure ou de marge directement dans le conteneur du panneau, modifie les dimensions globales et cela peut affecté le positionnement et l'animation. Pour éviter cela, appliquer des styles à l'enveloppe du contenu du panneau (.ui-pannel-inner).

Création de panneau responsive

Lorsque l'affichage push ou reveal est utilisé, le panneau pousse la page sur le côté lorsqu'il s'ouvre. Étant donné qu'une partie de la page est poussée hors de l'écran, le panneau est modale et doit être fermé pour interagir à nouveau avec le contenu de la page. Sur les grands écrans, vous voudrez peut-être que le panneau fonctionne plus comme une colonne pliable qui peut être ouverte et utilisée à côté de la page pour une meilleure utilisation de l'écran.

Pour que la page fonctionne à côté du panneau ouvert, il faut la redimensionner avec une largeur qui s'adapte à celle du panneau. Cela peut être fait uniquement avec des CSS en ajoutant une marge à gauche ou à droite égale à la largeur du panneau (17em) pour le contenu de la page pour forcer un redimensionnement. Deuxièmement, la couche invisible placée sur la page, pour le clic permettant de faire disparaitre le panneau, est cachée avec le CSS de sorte que vous puissiez cliquer sur la page et ne pas fermer le panneau.

Voici un exemple de ces règles enveloppés dans une media query qui applique uniquement ce comportement au dessus de 35em (560px) :


@media (min-width:35em){

	/* wrap on wide viewports once open */
	
	.ui-page-panel-open .ui-panel-content-fixed-toolbar-display-push.ui-panel-content-fixed-toolbar-position-left,
	.ui-page-panel-open .ui-panel-content-fixed-toolbar-display-reveal.ui-panel-content-fixed-toolbar-position-left,
	.ui-page-panel-open .ui-panel-content-wrap-display-push.ui-panel-content-wrap-position-left,
	.ui-page-panel-open .ui-panel-content-wrap-display-reveal.ui-panel-content-wrap-position-left {
		margin-right: 17em;
	}
	.ui-page-panel-open .ui-panel-content-fixed-toolbar-display-push.ui-panel-content-fixed-toolbar-position-right,
	.ui-page-panel-open .ui-panel-content-fixed-toolbar-display-reveal.ui-panel-content-fixed-toolbar-position-right,
	.ui-page-panel-open .ui-panel-content-wrap-display-push.ui-panel-content-wrap-position-right,
	.ui-page-panel-open .ui-panel-content-wrap-display-reveal.ui-panel-content-wrap-position-right {
		margin-left: 17em;
	}
	.ui-page-panel-open .ui-panel-content-fixed-toolbar-display-push,
	.ui-page-panel-open .ui-panel-content-fixed-toolbar-display-reveal {
		width: auto;
	}
	
	/* disable "dismiss" on wide viewports */
	.ui-panel-dismiss {
		display: none;
	}
	/* same as the above but for panels with display mode "push" only */
	.ui-panel-dismiss-display-push {
		display: none;
	}

}

L'application d'un point d'arrêt prédéfini

Incluez dans le style du widget un point d'arrêt prédéfini pour que ce comportement se déclenche au 55em (880px). Ce point d'arrêt n'est pas appliquée par défaut afin de rendre plus facile l'écriture des points d'arrêt personnalisées qui fonctionnent mieux pour votre contenu et votre design. Pour appliquer le point d'arrêt prédéfini, ajoutez la classe ui-responsive-panel au conteneur de la page (pas au panneau). Voir un exemple de panneau responsive.
