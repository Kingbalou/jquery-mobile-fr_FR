Le menu de sélection est basée sur un élément select natif, qui est caché de la vue et remplacé par un bouton de sélection avec un style personnalisé qui correspond à l'aspect et à la convivialité du framework jQuery Mobile. Le menu de sélection est ARIA-enabled et accessible par le clavier sur un ordinateur de bureau.

Par défaut, le framework s'appuie sur les options du menu natif de l'OS pour l'utiliser avec le bouton personnalisé. Lorsque le bouton est cliqué, le menu natif de l'OS va s'ouvrir. Quand une valeur est sélectionnée et que le menu se ferme, le texte du bouton personnalisé est mis à jour pour correspondre à la valeur sélectionnée.

Pour ajouter un menu de sélection sur votre page, commencez avec un élément standard select peuplé avec un ensemble d'élément option. Réglez l'attribut for de label pour faire correspondre l'ID du select ainsi ils sont sémantiquement liés. Il est possible de cacher le label d'accessibilité si il n'est pas souhaitée dans la mise en page, mais nous avons besoin qu'il soit présent dans le balisage pour des raisons de sémantique et d'accessibilité.

Le framework trouvera tous les éléments select et les améliorera automatiquement dans des menus de sélection, pas besoin d'appliquer l'attribut data-role. Pour éviter l'amélioration automatique d'un select, ajouter l'attribut data-role="none" au select.


<label for="select-choice-0" class="select">Mode de livraison :</label>
<select name="select-choice-0" id="select-choice-1">
   <option value="standard">Lent : 7 jours</option>
   <option value="rush">Rapide : 3 jours</option>
   <option value="express">Express : le lendemain</option>
   <option value="overnight">De nuit</option>
</select>

Cela va produire un menu de sélection de base. Les styles par défaut définissent la largeur de champ de saisie à 100% du conteneur parent et empile les labels sur une ligne distincte.

Optionnellement, enveloppez-les dans un conteneur avec l'attribut data-role="fieldcontain" afin de les grouper visuellement dans un formulaire plus long.


<div data-role="fieldcontain">
   <label for="select-choice-1" class="select">Mode de livraison :</label>
   <select name="select-choice-1" id="select-choice-1">
      <option value="standard">Lent : 7 jours</option>
      <option value="rush">Rapide : 3 jours</option>
      <option value="express">Express : le lendemain</option>
      <option value="overnight">De nuit</option>
   </select>
</div>

Zones de sélection regroupées verticalement

Pour créer un ensemble groupé de zones de sélection, ajoutez d'abord select et un label correspondant. Réglez l'attribut for de label pour faire correspondre l'ID du select ainsi ils sont sémantiquement liés.

Parce que l'élément label sera associé à chaque zonde de sélection, nous recommandons l'enveloppement des sélections dans un élément fieldset qui a une legend qui agit comme un label combiné pour les saisies groupées.

Enfin, on a besoin d'envelopper le fieldset dans un div avec l'attribut data-role="controlgroup", de sorte qu'il puisse être stylisé comme un groupe.

	
<div data-role="fieldcontain">
<fieldset data-role="controlgroup">
	<legend>Date de naissance :</legend>

    <label for="select-choice-month">Month</label>
<select name="select-choice-month" id="select-choice-month">
	<option>Mois</option>
	<option value="jan">Janvier</option>
	<!-- etc. -->
</select>

	<label for="select-choice-day">Day</label>
<select name="select-choice-day" id="select-choice-day">
	<option>Jour</option>
	<option value="1">1</option>
	<!-- etc. -->
</select>

<label for="select-choice-year">Year</label>
<select name="select-choice-year" id="select-choice-year">
	<option>Année</option>
	<option value="2011">2011</option>
	<!-- etc. -->
</select>
</fieldset>
</div>

Zones de sélection regroupées horizontalement

Les zones de sélection peuvent aussi être utilisées pour des ensembles groupés avec plus d'une des sélections connexes. Pour faire un ensemble de bouton horizontal, ajoutez data-type="horizontal" au fieldset. Notez que les boutons qui déclenchent les sélections seront redimensionner en fonction de la valeur de l'option actuellement sélectionnée. Notez que pour les navigateurs sans support pour display: inline-block; regroupera les sélections à la verticale, comme ci-dessus.

<fieldset data-role="controlgroup" data-type="horizontal">

Utilisation des menus personnalisés

the framework builds a custom menu based on the select element's list of options. Nous vous recommandons d'utiliser un menu personnalisé lorsque les sélections multiples sont nécessaires, comme, ou quand le menu lui-même doit être stylé avec CSS.

Vous pouvez éventuellement utiliser des menus de sélection avec des styles personnalisés au lieu du menu OS natif. Le menu personnalisé supporte les options désactivées et la sélection multiple (alors que le support natif de l'OS du mobile pour les deux est incompatible), cela apporte une façon élégante de gérer les valeurs placeholder et reconstitue la fonctionnalité manquante sur certaines plateformes telles que le support de optgroup sur Android (tout est expliqué ci-dessous). En outre, le framework applique le thème du bouton personnalisé pour le menu afin de mieux correspondre à l'aspect et la convivialité, et fournit une cohérence visuelle entre les plateformes. Enfin, les menus personnalisés présentent souvent mieux sur les navigateurs de bureau, car les menus de bureau natifs sont plus petits que leurs homologues mobiles et ont tendance à être disproportionné.

Gardez à l'esprit qu'il y a une surcharge impliquée dans l'analyse du select natif afin de construire un menu personnalisé. Si il y a beaucoup de sélection sur une page ou une sélection avec une longue liste d'options, ceci peut affecter les performances de la page, donc nous vous recommandons d'utiliser des menus personnalisés avec parcimonie.

Pour utiliser les menus personnalisés sur un select particulier, ajoutez simplement l'attribut data-native-menu="false". Alternativement, on peut aussi par programme définir l'option de configuration nativeMenu du menu de sélection à false dans la fonction de rappel liée à l'événement mobileinit pour obtenir le même effet. Cela sera globalement fait sur tous les select afin d'utiliser le menu personnalisé par défaut. Les éléments suivants doivent être inclus dans la page après que jQuery soit chargé mais avant que jQuery mobile soit chargé.

$(document).bind('mobileinit',function(){
   $.mobile.selectmenu.prototype.options.nativeMenu = false;
});

Cela nous permet d'utiliser le défilement natif inclus sur l'appareil pour se déplacer dans une longue liste. Le texte à l'intérieur du label est utilisé comme titre de cette page.

Options placeholder

Il est commun pour les développeurs d'inclure une option "null" dans leur élément select pour forcer un utilisateur à choisir une option. Si une option placeholder est présente dans votre balisage, jQuery Mobile le cachera dans le menu qui apparait, ne montrant que les choix valide pour l'utilisateur, et affichera le texte placeholder à l'intérieur du menu, comme une entête. L'option placeholder est ajoutée lorsque le framework trouve :

  • Une option sans attribut de valeur (ou une valeur d'attribut vide)
  • Une option sans noeud de texte
  • Une option avec l'attribut data-placeholder="true". (Cela vous permet d'utiliser une option qui a une valeur et un textnode comme une option placeholder).

Vous pouvez désactiver cette fonction via l'option hidePlaceholderMenuItemsdu plugin selectmenu, comme ceci :

	
$.mobile.selectmenu.prototype.options.hidePlaceholderMenuItems = false;

Sélection multiple

Si l'attribut multiple est présent dans votre balisage, jQuery Mobile renforcera l'élément avec quelques considérations supplémentaires :

  • Un élément entête sera créé à l'intérieur du menu et l'affichage du texte du placeholder et d'un bouton de fermeture.
  • En cliquant sur un élément à l'intérieur de la fenêtre, cela ne fermera pas le widget.
  • Une icône fantôme et non cochée apparaîtra à côté de chaque élément non sélectionné. Lorsque l'élément est sélectionné, l'icône va changer en une case à cochée. Aucune icône apparaît dans le cas d'une boîte d'une sélection unique.
  • Une fois que plus de deux éléments sont sélectionnés, un élément compteur avec le nombre total d'éléments sélectionnés apparait à l'intérieur du bouton.
  • Le texte de chaque élément sélectionné apparait à l'intérieur du bouton comme une liste. Si le bouton n'est pas assez large pour afficher toute la liste, il est tronqué avec une ellipse.
  • Si aucun élément n'est sélectionné, le texte du bouton sera par défaut le texte du placeholder.
  • Si aucun élément placeholder existe, le texte du bouton sera par défaut vide et l'entête apparaîtra avec juste un bouton de fermeture. Parce que ce n'est pas une expérience conviviale, nous vous recommandons de toujours spécifier un élément placeholder lorsque vous utilisez des boîtes de sélection multiple.

Quand une sélection est assez longue, le menu sera ouvert dans une nouvelle page, le texte du placeholder est affiché dans le bouton lorsqu'aucuns éléments sont sélectionnés, et le texte du label est affiché dans l'entête du menu. Cela diffère des petits menus où le texte du placeholder est affiché à la fois sur le bouton et l'entête, et dans le cas d'une pleine page avec une sélection unique où le texte du placeholder n'est pas du tout utilisé.

Appel du plugin menu de sélection

Le plugin menu de sélection initialisera automatiquement n'importe quelle page contenant le menu de sélection, il n'y a pas besoin d'un attribut data-role dans le balisage. Cependant, vous pouvez appeler directement le plugin de menu de sélection sur n'importe quel sélecteur, comme n'importe quel plugin normal de jQuery :


$('select').selectmenu();
