API des boutons

Les boutons sont des widgets de base dans jQuery Mobile et sont utilisés dans beaucoup d'autres plugins. Le balisage du bouton est flexible et peut être créé depuis les liens ou les boutons de formulaires. Sauter vers une section  

Balisage

Utilisez les liens d'ancrage (éléments a) pour marquer les boutons de navigation et les éléments input ou button pour la soumission de formulaire. Par défaut, tous les boutons dans le contenu du corps sont stylisés comme élément de bloc-niveau afin qu'ils remplissent la largeur de l'écran :

Aligné

Si vous avez plusieurs boutons qui doivent se positionner côte à côte sur la même ligne, ajoutez l'attribut data-inline="true" à chaque bouton. Cela stylisera les boutons avec la largeur de leur contenu ainsi les boutons seront positionner sur la même ligne.

Thème

Les boutons peuvent être attribuées manuellement à n'importe quels nuanciers de couleur des boutons à partir du thème pour ajouter du contraste visuel avec le conteneur, il faut le positionner à l'intérieur en ajoutant l'attribut data-theme sur la balise bouton et une lettre précisant le nuancier. Quand un lien est ajouté à un conteneur, il lui est attribué automatiquement une lettre de nuancier de thème qui correspond à la barre ou la boîte de contenu du son parent pour visuellement intégrer le bouton dans le conteneur parent, comme un caméléon.

Mini

Pour avoir une version plus compacte, qui est utile dans les barres d'outils et les espaces restreints, ajoutez l'attribut data-mini="true" au bouton pour créer une version mini.

Position d'icône

Par défaut, toutes les icônes dans les boutons sont placés à la gauche du texte du bouton. Ce défaut peut être modifiée en utilisant l'attribut data-iconpos pour définir l'icône à la droite (right), au-dessus (top) ou en dessous (bottom) du texte. Vous pouvez également créer un bouton icône seule, en définissant l'attribut data-iconpos à notext. Le plugin bouton masquera le texte à l'écran, mais l'ajoutera comme un attribut title sur le lien pour fournir un contexte pour les lecteurs d'écran et les appareils qui supportent les infobulles.

Groupe vertical

Pour regrouper, visuellement un ensemble de boutons pour former un seul bloc, enveloppez un ensemble de boutons dans un container avec l'attribut data-role="controlgroup" — le framework créera un groupe de boutons vertical, enlèvera toutes les marges et les ombres portées entre les boutons et laissera seulement l'arrondi du premier et dernier bouton de la série pour créer un effet de regroupement.

Groupe horizontal

En ajoutant l'attribut data-type="horizontal" au container controlgroup, vous pouvez transformer en un groupe horizontal qui positionne les boutons côte à côte et définir une largeur pour être assez grand pour accueillir le contenu.

Coins & Ombres

Désactivé

Les boutons et les saisie du formulaire peuvent être désactivé via l'attribut disabled. Les liens stylisés comme les boutons, ont les mêmes options visuelles comme sur les boutons d'un formulaire, mais ils ne font pas partie du plugin bouton ainsi les méthodes de bouton de formulaire (activer, désactiver, de rafraîchissement) ne sont pas pris en charge. Si vous avez besoin de désactiver un lien basé sur le bouton (ou tout autre élément), il est possible de lui appliquer la classe ui-disabled avec du JavaScript afin de réaliser le même effet.
