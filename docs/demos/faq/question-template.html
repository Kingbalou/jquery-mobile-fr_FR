Question :

Pourquoi mes styles CSS ne s'appliquent pas correctement à un widget ?

Réponse :

C'est probablement parce que l'élément que vous voyez dans le balisage (bouton, élément de formulaire, etc) n'est pas le même après les améliorations qui ont été appliquées. En vue de styliser les éléments et d'ajouter le comportement, jQuery Mobile modifie le balisage d'origine avec des classes supplémentaires et des éléments qui pourraient avoir une incidence sur vos sélecteurs CSS. Utilisez un inspecteur Web pour afficher le balisage après amélioration pour trouver les sélecteurs qui doivent être utiliser.

Regardons un exemple pour illustrer les améliorations. Voici le balisage de départ pour un bouton :


// Bouton avant amélioration
<button>Un élément bouton</button>

Après l'amélioration, le balisage du bouton a été modifiée par le framework pour ajouter des classes et envelopper le contenu pour styliser :


// Bouton après amélioration
<div data-corners="true" data-shadow="true" data-iconshadow="true" data-wrapperels="span" data-icon="null" data-iconpos="null" data-theme="c" class="ui-btn ui-shadow ui-btn-corner-all ui-btn-up-c" aria-disabled="false">
	<span class="ui-btn-inner ui-btn-corner-all">
		<span class="ui-btn-text">Un élément bouton</span>
	</span>
	<button class="ui-btn-hidden" aria-disabled="false">Un élément bouton</button>
</div>

Comme vous pouvez le voir dans l'extrait de code ci-dessus le balisage a considérablement changé après l'amélioration. Le bouton d'origine que vous avez ajouté a été enveloppé dans un div et plusieurs spans ont été ajoutés. Le bouton d'origine que vous avez ajouté est désormais également masqué. Pour modifier le style visuel du bouton, vous devez cibler le nouveau balisage amélioré, pas le bouton que vous avez ajouté.
