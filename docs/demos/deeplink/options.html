Le plugin curseur a les options suivantes :

disabled boolean

par défaut : false

Définit l'état par défaut du curseur à désactivé si la valeur est à "true".

$('.selector').slider({ disabled: true });
highlight boolean

par défaut : false

Définit que la partie remplie sur la piste à gauche du curseur est active lorsqu'il est réglé sur "true".

$('.selector').slider({ highlight: true });
initSelector CSS selector string

default: "input[type='range'], :jqmData(type='range'), :jqmData(role='slider')"

Ceci est utilisé pour définir les sélecteurs (types d'éléments, les rôles de données, etc) qui seront automatiquement initialisés comme des curseurs. Pour changer les éléments qui sont initialisés, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
   $.mobile.slider.prototype.options.initSelector = ".myslider";
});
mini boolean

par défaut : false

Définit la taille de l'élément pour qu'il soit plus compact : une version mini. Cette option est également définie comme un attribut de données : data-mini="true"

$('.selector').slider({ mini: true });
theme string

par défaut : null, hérité d'un parent

Définit le jeu de couleurs (nuancier) pour toutes les instances de ce widget. Il accepte une seule lettre de a à z qui correspond aux nuanciers inclus dans votre thème. Par défaut, il hérite de la même couleur de nuancier que le conteneur parent qui le possède s'il n'est pas explicitement défini. Cette option est également définie comme un attribut de données : data-theme="a"

$('.selector').slider({ theme: "a" });
trackTheme string

par défaut : null, hérité d'un parent

Définit le jeu de couleurs (nuancier) spécifiquement pour la piste du curseur. Il accepte une seule lettre de a à z qui correspond aux nuanciers inclus dans votre thème.

$('.selector').slider({ trackTheme: "a" });

Cette option peut être substituée dans le balisage en assignant un attribut de données à la zone de saisie, par exemple, data-track-theme="a"
