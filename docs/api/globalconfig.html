Travailler avec l'Auto-initialisation de jQuery Mobile

Contrairement aux autres projets jQuery, tels que jQuery et jQuery UI, jQuery Mobile applique automatiquement de nombreuses améliorations de balisage dès le chargement (bien avant que l'évènement document.ready se déclenche). Ces améliorations sont appliquées en fonction de la configuration par défaut de jQuery Mobile, qui est conçu pour fonctionner avec des scénarios communs, cependant cela peut ou ne peut pas correspondre à vos besoins particuliers. Heureusement, ces paramètres sont faciles à configurer.

L'événement mobileinit

Lorsque jQuery mobile commence à s'exécuter, il déclenche un événement mobileinit sur ​​l'objet document, que vous pouvez lier pour écraser les valeurs par défaut de jQuery Mobile.

				
$(document).bind("mobileinit", function(){
  //Appliquer des remplacements ici
});

Comme l'événement mobileinit est déclenché immédiatement après l'exécution, vous devez lier votre gestionnaire d'événements avant que jQuery Mobile soit chargé. Ainsi, nous recommandons la liaison de vos fichiers JavaScript dans l'ordre suivant :

				
<script src="jquery.js"></script>
<script src="script-perso.js"></script>
<script src="jquery-mobile.js"></script>

Au sein de cet événement, vous pouvez configurer les valeurs par défaut, soit en étendant l'objet $.mobile soit à l'aide de la méthode $.extend de jQuery :

				
$(document).bind("mobileinit", function(){
  $.extend(  $.mobile , {
    foo: bar
  });
});

... ou en les définissant individuellement :

				
$(document).bind("mobileinit", function(){
  $.mobile.foo = bar;
});

Pour prévisualiser rapidement ces options de configuration globale à l'œuvre, consultez les pages de test de configuration.

Options configurables

Les valeurs par défaut suivantes sont configurables via l'objet $.mobile :

ns string, par défaut : ""
L'espace de nom utilisé dans les attributs data-, par exemple data-role, peut être mis avec n'importe quelle valeur, y compris une chaîne vide, qui est celle par défaut. A l'utilisation, ce sera plus clair si vous incluez un tiret, comme "mynamespace-" qui correspond à data-mynamespace-foo="...".

REMARQUE : si vous utilisez des espaces de nommage data-, vous aurez besoin de mettre à jour/écraser manuellement un sélecteur dans le CSS du thème. Les sélecteurs de donnée suivants devront intégrer l'espace de nommage que vous utilisez : 


.ui-mobile [data-mynamespace-role=page], .ui-mobile [data-mynamespace-role=dialog], .ui-page { ...

autoInitializePage boolean, par défaut : true
Quand le DOM est prêt, le framework devrait automatiquement appeler $.mobile.initializePage. Si c'est à false, la page ne sera pas initialisée et et sera visuellement cachée jusqu'à ce que $.mobile.initializePage soit appelé manuellement.
subPageUrlKey string, par défaut : "ui-page"
Le paramètre d'url est utilisé pour le référencement des sous-pages de widget-généré (comme ceux générés par des listes imbriquées). Cela se traduit par example.html&ui-page=subpageIdentifier. Le segment hash avant &ui-page= est utilisé par le framework pour faire une requête Ajax vers l'URL où se trouve la sous-page.
activePageClass string, par défaut, "ui-page-active"
La classe attribuée à la page actuellement visible et pendant les transitions
activeBtnClass string, par défaut: "ui-btn-active"
La classe utilisée pour le statut du bouton "actif" depuis le framework de CSS.
ajaxEnabled boolean, par défaut: true
jQuery Mobile gérera automatiquement les clics sur les liens et soumettra le formulaire grâce à Ajax, quand cela est possible. Si c'est à false, l'écoute de l'url hash sera désactivé, ainsi les urls se chargeront comme des requêtes HTTP régulières.
linkBindingEnabled boolean, par défaut : true
jQuery Mobile liera automatiquement les clics sur les balises d'ancrage dans votre document. Le paramétrage de ces options à false empêchera tous les ancrages à l'ajout d'including gérant le clic de l'état du bouton actif et de lien alternatif flouté. Cela ne devrait être utilisé que lorsque l'on tente de déléguer la gestion du clic à une autre bibliothèque ou un code personnalisé.
hashListeningEnabled boolean, par défaut : true
jQuery Mobile écoutera et gérera automatiquement les changements vers location.hash. La désactivation empêchera jQuery Mobile de manipuler les changements de hash, ce qui vous permet de le gérer vous-même, ou tout simplement pour utiliser de simples liens dans un document qui défilent vers un ID particulier.
pushStateEnabled boolean, par défaut : true
Amélioration pour utiliser history.replaceState dans les navigateurs pris en charge, pour convertir les URL basées sur le hash Ajax dans le chemin du document complet. Notez que nous recommandons la désactivation de cette fonction si l'Ajax est désactivé ou si l'utilisation intensive des liens externes est utilisée.
defaultPageTransition string, par défaut : 'slide'
Définit la transition par défaut pour les changements de page lors de l'utilisation d'Ajax. Définissez à 'none' pour qu'il n'y ait aucune transition par défaut.
touchOverflowEnabled boolean, par défaut : false
Active le lissage des transitions de page et des barres d'outils fixes dans les périphériques qui supportent les propriétés CSS overflow: et overflow-scrolling: touch; .
defaultDialogTransition string, par défaut : 'pop'
Définit la transition par défaut pour les changements de boîte de dialogue qui utilise Ajax. Définissez à 'none' pour qu'il n'y ait aucune transition par défaut.
minScrollBack string, par défaut : 250
La distance minimale de défilement qui restera en mémoire lors du retour à une page.
loadingMessage string, par défaut : "loading"
Définit le texte qui apparaît quand une page se charge. Si c'est à false, le message n'apparaîtra pas du tout.
loadingMessageTheme string, par défaut : "a"
Définit le thème utilisé pour les boîtes de message de chargement, lorsque le texte est visible.
pageLoadErrorMessage string, par défaut: "Error Loading Page"
Définit le texte qui apparaît quand une page ne se charge pas grâce à l'Ajax.
pageLoadErrorMessageTheme string, par défaut : "e"
Définit le thème utilisé pour les boîtes de message d'erreur.
loadingMessageTextVisible string, par défaut : false
Si le texte doit être visible lorsque le message de chargement est indiqué. (Remarque : Actuellement, le texte est toujours visible pour les erreurs de chargement)
gradeA fonction qui retourne un boolean, par défaut : Une fonction retournant la valeur de $.support.mediaquery
N'importe quelles conditions de support qui doivent être respectées pour passer.
