Classes d'aide Media Query

Remarque : Cette fonction était dépréciée en version bêta, et est supprimée en 1.0rc1. Nous vous recommandons d'utiliser les Media Queries CSS3 à la place. Afin de soutenir les anciennes versions d'Internet Explorer, consultez respond.js,Un polyfill rapide et léger pour les Media Queries min/max-width CSS3.

Si vous avez encore besoin de cette fonctionnalité, vous pouvez trouver le code ici: jquery.mobile.media.classes.js

jQuery Mobile ajoute les classes à l'élément HTML qui imitent l'orientation du navigateur et les Media Queries CSS communes min/max-width. Ces classes sont mises à jour lors de load, resize et orientationchange, vous permettant d'éteindre ces classes dans votre CSS, créer des mises en page adaptif - même dans les navigateurs qui ne supportent pas les medias queries !

Classes d'orientation

L'élément HTML aura toujours soit une classe "portrait" ou "landscape", selon l'orientation du navigateur ou du périphérique. Vous pouvez utiliser ces derniers dans votre CSS comme ceci :

			
.portrait {
	/* changements orientation portrait rendez-vous ici ! */
}
.landscape {
	/* changements orientation paysage rendez-vous ici  */
}

Classes breakpoint de largeur Min/Max

Par défaut, nous créons les classes breakpoint Min/Max de largeur suivantes : 320,480,768,1024. Cela se traduit par des classes qui ressemble à ceci :"min-width-320px", "max-width-480px", et sont destinés à être utilisé comme un remplacement (ou en plus) aux media query équivalents.

			
.myelement { 
	float: none;
}			
.min-width-480px .myelement {
	float: left;
}

De nombreux plugins jQuery Mobile se base sur ses breakpoints de largeur. Par exemple, les éléments du formulaire flottent à côté de leurs libellés lorsque le navigateur est plus large que 480 pixels. Le CSS pour supporter ce comportement pour les saisies de texte de formulaire ressemble à ceci :

			
label.ui-input-text { 
	display: block; 
}
.min-width-480px label.ui-input-text { 
	display: inline-block; 
}

Ajout de breakpoints de largeur

Pour utiliser les breakpoints de largeur, jQuery Mobile propose la fonction $.mobile.addResolutionBreakpoints , qui accepte soit un seul numéro ou un tableau de nombre qui seront ajoutés aux breakpoints min/max à chaque fois qu'ils s'appliquent.

			
//ajoute une classe min/max de 1200 pixel de large			
$.mobile.addResolutionBreakpoints(1200);

//ajoute une classe min/max de 1200 et 1440 pixel de large
$.mobile.addResolutionBreakpoints([1200, 1440]);

Exécution des Media Queries

jQuery Mobile fournit une fonction qui vous permet de tester si une Media Query CSS s'applique. Appelez simplement $.mobile.media() et passer un type de media ou une requête. Si le navigateur supporte le type ou la requête, il s'applique directement et la fonction retournera true. Sinon, elle retournera false.

			
//test pour le type de média screen
$.mobile.media("screen");

//test  pour le media query min-width
$.mobile.media("screen and (min-width: 480px)");

//test pour l'affichage d'iOS retina
$.mobile.media("screen and (-webkit-min-device-pixel-ratio: 2)");
