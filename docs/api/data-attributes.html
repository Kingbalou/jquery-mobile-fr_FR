Référence de l'attribut data-

Le framework jQuery Mobile utilise les attributs HTML5 data- pour permettre l'initialisation et la configuration des widgets basé sur le balisage. Ces attributs sont complètement optionnels, en appelant les plugins manuellement et en passant les options directement, cela est également supporté. Pour éviter les conflits de noms avec d'autres plugins ou des frameworks qui utilisent également les attributs data-, définissez un espace de nommage personnalisé en modifiant l'option globale ns.

Bouton

Liens avec data-role="button". Les boutons basés sur l'input et les éléments button sont améliorés automatiquement, donc pas besoin de data-role

data-corners true | false
data-icon home | delete | plus | arrow-u | arrow-d | check | gear | grid | star | custom | arrow-r | arrow-l | minus | refresh | forward | back | alert | info | search
data-iconpos left | right | top | bottom | notext
data-iconshadow true | false
data-inline true | false
data-mini true | false - version d'une taille compacte
data-shadow true | false
data-theme Lettre de l'échantillon (a-z)

Plusieurs boutons peuvent être enveloppés dans un conteneur avec l'attribut data-role="controlgroup" pour les regrouper verticalement. Ajoutez l'attribut data-type="horizontal" pour que les boutons soit côte à côte.

Amélioration

Conteneur avec data-enhance="false" ou data-ajax="false"

data-enhance true | false
data-ajax true | false

Tous les éléments du DOM au sein d'un conteneur avec data-enhance="false", sauf pour les éléments data-role="page|dialog", seront ignorés lors de l'amélioration initiale et des événements create ultérieurs à condition que le $.mobile.ignoreContentEnabled soit défini avant l'amélioration (par exemple dans un bind de mobileinit).

Les éléments d'un lien ou d'un formulaire à l'intérieur des conteneurs data-enhance="false" seront ignorés par la fonctionnalité de navigation du framework lorsque $.mobile.ignoreContentEnabled est défini à true.
