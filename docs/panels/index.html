Balisage du panneau basique

Les panneaux peuvent être utilisés pour plusieurs besoins dans un site ou une application. Ils sont conçus pour être aussi ouvert que possible afin de simplifier la création des menus, des colonnes pliables, des tiroirs, des panneaux inspecteurs et plus encore. Il s'agit simplement de blocs qui peuvent se positionner à la gauche ou la droite du contenu de page et apparaître de différentes manières, lorsqu'ils sont activés par un lien.

Panneau par défaut

Un panneau est constitué d'un conteneur avec l'attribut data-role="panel" et un ID unique. Cet ID sera référencé par un lien ou un bouton pour ouvrir et fermer le panneau.

Un panneau doit être enfant des éléments header, content et footer à l'intérieur de la page jQuery Mobile. Vous pouvez ajouter le balisage du panneau soit avant ou après ces éléments, mais pas entre les deux. Un panneau ne peut pas être placé à l'extérieur d'une page, mais cette contrainte sera supprimée dans une future version.

Voici un exemple du panneau avant le header, content et footer dans l'ordre de la source :


<div data-role="page">			

	<div data-role="panel" id="mypanel">
		<!-- le contenu du panneau se place ici -->
	</div><!-- /panel -->

	<!-- header -->
	<!-- content -->
	<!-- footer -->

</div><!-- page -->

Alternativement, il est possible d'ajouter le balisage du panneau après le header, content et footer dans l'ordre de la source, juste avant la fin du conteneur de page. Selon l'endroit où vous placez le balisage du panneau, dépendra de la façon dont vous voulez que contenu de la page soit lu pour les personnes qui se trouvent sur la page dans un dispositif de grade C (HTML seulement) ou d'un lecteur d'écran.

Ouverture d'un panneau

L'aperçu du panneau est déclenché par un lien quelque part sur ​​la page ou en appelant directement la méthode open du panneau.

$( "#idofpanel" ).panel( "open" , optionsHash );

<a href="#mypanel" data-rel="panel">Ouvrir le panneau</a>

Ceci créera un lien qui ouvre notre panneau avec les options par défaut. Les valeurs par défaut placent le panneau sur la gauche et pousse la page quand il s'ouvre (mode reveal).

Pour contrôler un panneau depuis un lien, pointez le href à la référence du ID du panneau que vous voulez faire apparaitre (mypanel dans l'exemple ci-dessus), et ajoutez l'attribut data-rel="panel". Ceci indique au framework de lier le lien au panneau. Ce lien permet de basculer l'affichage de la popup afin d'ouvrir le panneau en cliquant dessus et de le fermer en cliquant à nouveau dessus.

Ouvrir le panneau

Lorsque vous utilisez des balises pour contrôler les panneaux, vous ne pouvez avoir à la fois qu'un seul panneau ouvert. En cliquant sur un lien pour ouvrir un deuxième panneau alors qu'un panneau est déjà ouvert, cela fermera automatiquement le premier. Ceci est fait pour maintenir la simplicité d'une configuration de balisage unique.

Cela dit, il est possible d'ouvrir plusieurs panneaux à la fois par programmation :


$( "#idofpanel" )
.panel( "open" , optionsHash )
.then( function( options ){
	$( "#idofpanel2" ).panel( "open" , options );
});

Fermeture d'un panneau

En cliquant sur ​​le lien qui a ouvert le panneau ou en tapant sur ​​la touche Echap, cela ferme le panneau. Un panneau peut également être fermé en appelant directement la méthode close du panneau.

$( "#idofpanel" ).panel( "close" );

Par défaut, en cliquant en dehors du panneau sur la page, cela va également fermer le panneau. Ce comportement peut être contrôlé via l'option dismissible.

Il est fréquent d'ajouter un bouton de fermeture à l'intérieur du panneau. Pour ajouter un lien qui va fermer un panneau, ajoutez un lien quelque part à l'intérieur du panneau qui a l'attribut data-rel="close". Cela dira au framework de fermer ce panneau.


<a href="#mypanel" data-rel="close">Fermer le panneau</a>

Paramétrage du mode d'affichage du panneau

Les panneaux peuvent se présenter sous trois manières différentes. Le mode d'affichage par défaut est reveal mais cela peut être configuré sur chaque lien en ajoutant l'attribut data-display et en spécifiant un mode d'affichage.

reveal (par défaut)
Le panneau se déplace par-dessus la zone d'affichage et partiellement il occulte une partie de la page quand il la pousse hors de la vue.
push
Le panneau se comporte exactement comme "pan" sur des largeurs plus étroites mais sur des fenêtres plus larges (ci-dessus 600 pixels), le contenu se rétrécie pour faire de la place à l'écran de sorte qu'il puisse rester ouvert en permanence. Notez que pour des écrans plus larges, l'option dismissible sera définit à false de sorte que vous pouvez interagir avec le panneau et la page.
overlay
Le panneau glisse sur le contenu comme un menu standard.

Réglage de la position du panneau

Les panneaux peuvent s'ouvrir à gauche ou à droite de la page, ceci est contrôlé par le lien qui les ouvre. La position par défaut est left (gauche) mais cela peut être configuré sur chaque lien en ajoutant l'attribut data-position et en spécifiant une position d'affichage.

left (par défaut)
Le panneau va glisser à partir du bord gauche de la page.
right
Le panneau va glisser à partir du bord droit de la page.

Paramétrage pour annuler le comportement

En cliquant à l'extérieur d'un panneau ouvert, cela ferme par défaut le panneau. Ce comportement peut être évité en ajoutant l'attribut data-dismissible au lien et en spécifiant true ou false.

true (par défaut)
Le panneau s'auto-fermera lorsqu'un utilisateur touchera l'extérieur du panneau.
false
Le panneau ne se fermera pas lorsqu'un utilisateur touchera l'extérieur du panneau. Il est recommandé pour le mode d'affichage push pour permettre à un utilisateur d'interagir avec la page lorsque le panneau est affiché sur des grandes largeurs.

Options

classes.panel par défaut : "ui-panel"
L'option classes a deux propriétés qui définissent les noms de classes structurelles que le plugin utilise. Ce n'est que configurable via JavaScript car il attend une valeur littérale d'objet.
dismissible par défaut : true
Définit si le panneau peut être fermé en cliquant sur ​​l'extérieur sur la page. Également disponible via l'attribut data-dismissible sur le lien qui ouvre le panneau.
display par défaut : "reveal"
La relation entre le panneau et le contenu de la page. Peut être soit poussé sur la page ("reveal"), soit rétrécir le contenu pour s'adapter au contenu du panneau sous forme de colonne ("push"), ou soit superposer le contenu ("overlay"). Également disponible via l'attribut data-display sur le lien qui ouvre le panneau.
initSelector par défaut : ":jqmData(role='panel')"
Le rôle (et ou un sélecteur jQuery valide) utilisé pour déclencher l'auto-initialisation du widget panneau.
position par défaut : "left"
Le côté de l'écran, où le panneau apparaît. Les valeurs peuvent être "left" ou "right. Également disponible via l'attribut data-position sur le lien qui ouvre le panneau.
theme par défaut : null
Nuancier du thème pour le panneau. Peut être une lettre de nuancier valable dans votre thème (a-z). Également disponible via l'attribut data-theme sur le conteneur du panneau.

Méthodes

close
Ferme le panneau. .
$( myPanel ).panel( "close" );
open
Ouvre le panneau. 
$( myPanel ).panel( "open" );
refresh
Re-améliore tout le nouveau balisage dans le panneau. 
$( myPanel ).panel( "refresh" );
toggle
Active ou désactive la visibilité du panneau de sorte qu'il ouvrira un panneau fermé ou fermera un panneau ouvert. 
$( myPanel ).panel( "toggle" );

Événements

create Type : panelcreate
Déclenché lorsque le panneau est créé.
