Structure de barre de pied de page

La barre footer a la même structure de base qu'une entête sauf qu'elle utilise l'attribut data-role avec la valeur footer. 

<div data-role="footer"> 
	<h4>Contenu du pied de page</h4> 
</div>

La barre d'outils footer sera thématisé avec le nuancier "a" par défaut (noir dans le thème par défaut) mais vous pouvez facilement définir une couleur de nuancier du thème.

Contenu de pied de page

La page "pied de page" est très similaire à l'entête en termes d'options et de configuration. Les principales différences sont que le pied de page est conçu pour être moins structuré que l'entête, afin de permettre plus de flexibilité, de sorte que le framework ne place pas automatiquement des boutons à gauche ou à droite selon l'ordre du source comme cela est fait dans l'entête. Étant donné que les pieds de page n'ont pas les mêmes prétentions de balisage prescriptif comme les entêtes, utilisez la mise en page grids ou des styles personnalisés pour réaliser la conception que vous voulez dans un pied de page.

Ajout de boutons

Tout lien ou balisage de button valide ajouté au pied de page sera automatiquement transformé en un bouton. Pour gagner de l'espace, les boutons dans les barres d'outils sont automatiquement mis avec un style en ligne de sorte que le bouton soit aussi large que le texte et l'icône qu'elle contient.

Par défaut les barres d'outils n'ont pas de espacement pour accueillir les barres de navigation et d'autres widgets. Pour inclure un espacement sur la barre, ajouter un class="ui-bar" au pied de page.


<div data-role="footer" class="ui-bar">
	<a href="index.html" data-role="button" data-icon="delete">Enlever</a>
	<a href="index.html" data-role="button" data-icon="plus">Ajouter</a>
	<a href="index.html" data-role="button" data-icon="arrow-u">Haut</a>
	<a href="index.html" data-role="button" data-icon="arrow-d">Bas</a>
</div>

Cela crée cette barre d'outils avec des boutons positionnés sur une rangée

Pieds de page persistants

Dans les situations où le pied de page est un élément de navigation globale, vous pouvez le voir apparaître fixe et en place entre les transitions de page. Ceci peut être accompli en utilisant la fonctionnalité persistante du pied de page inclus dans jQuery Mobile.

Pour faire un pied de page qui reste en place entre les transitions, ajoutez l'attributdata-id pour le pied de page de toutes les pages pertinentes et utilisez le même id pour chaque page. Par exemple, en ajoutant data-id="myfooter" à la page en cours et à la page cible, le framework gardera les ancres de pied de page au même endroit durant l'animation page. VEUILLEZ NOTER : Cet effet fonctionnera correctement seulement si l'entête et le pied de page des barres d'outils sont définis avec data-position="fixed" de sorte qu'ils soient visibles durant la transition.
