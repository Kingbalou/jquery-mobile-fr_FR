Barres d'outils fixées

Ceci est une démonstration des entêtes et des pieds de page "fixés" utilisés dans le framework jQuery Mobile. Le contenu de la page se déroule naturellement, nous permettant de profiter du défilement natif au lieu d'écrire une solution de contournement de faux défilement. Les divs de l'entête et du pied de page sont donc dans le flux du document, mais chaque fois qu'ils sont hors de l'affichage, le framework les re-positionnera dynamiquement dans la vue si le navigateur supporte cette fonctionnalité, sinon ils vont tout simplement rester en ligne.

Pour activer ce comportement sur ​​une entête ou un pied de page, ajoutez l'attribut data-position="fixed" au conteneur de la barre d'outils.

Appuyez pour choisir la visibilité

Pour choisir la visibilité des barres d'outils fixées, appuyez sur l'écran. Par exemple, si les barres d'outils fixées sont visibles, appuyez sur l'écran pour cacher les barres d'outils et pour profiter pleinement de l'écran complet pour le contenu. En tapant de nouveau cela ramènera l'affichage des barres d'outils.

Il est possible de fermer le robinet du comportement de bascule de la visibilité comme ceci :


$.mobile.fixedToolbars
   .setTouchToggleEnabled(false);

Mise à jour du positionnement de la barre d'outils

Si la hauteur de la page change, soit par l'injection dynamique de balisage, ou par des widgets qui cachent ou plient le contenu, cela peut rejeter le positionnement dynamique des barres d'outils. Pour demander manuellement à la barre d'outils de se repositionner, puis apparaitre en fondu, utilisez $.mobile.fixedToolbars.show();. Pour les faire apparaître immédiatement, sans le fondu :


$.mobile.fixedToolbars
   .show(true);

Il y a aussi un événement updatelayout qui peut être utilisé pour déclencher les barres d'outils pour se repositionner. Les développeurs qui créent des applications dynamiques qui injectent du contenu dans la page en cours peuvent également déclencher manuellement cet événement updatelayout pour assurer la mise à jour des composants sur la page en réponse à du nouveau contenu qui vient d'être ajouté. Cet événement est utilisé en interne dans les plugins de pliable et de filtre de liste et il est puissant parce qu'il n'est pas spécifique à la barre d'outils, n'importe quel widget peut être construit pour écouter l'événement updatelayout pour mettre à jour le widget dans la réponse.

Limitations connues

jQuery Mobile uses dynamically re-positioned toolbars for the fixed header effect because very few mobile browsers support the position:fixed CSS property. Although our fixed toolbar feature works fairly well, there are a number of technical limitations that can cause the toolbars to appear to scroll with the page. Most of these rendering issues are due to the fact many mobile platforms (iOS, Android, etc.) essentially take a static screenshot of the page and display this image during scrolling instead of the actual rendered HTML. This improves scrolling performance, but when scrolling happens quickly, the toolbars will be "burned" into the page screenshot before our script can hide them so they appear to scroll with the page. We have optimized this as much as we possibly can, but there are going to be situations where fixed toolbars won't work perfectly due to browser limitations, so this is important to note when considering whether to use this feature.

True fixed toolbars: touchOverflowEnabled

In order to achieve true fixed toolbars, a browser needs to either support position:fixed or overflow:auto. Fortunately, this support is coming to mobile platforms so we can achieve this with web standards. In jQuery Mobile, we have added a global feature called touchOverflowEnabled that leverages the overflow:auto CSS property on supported platforms like iOS5. When enabled, the framework wraps each page in a container with it's own internal scrolling. This allows us to position the toolbars outside the scrolling body so they truly stay fixed in place at all times. Learn more about this feature on the global options page or demo this feature (currently iOS5 only, other browsers will fall back to dynamically re-positioned fixed toolbars).
