Barres d'outils fixes

Ceci est une démonstration des entêtes et des pieds de page "fixes" utilisés dans le framework jQuery Mobile. Le contenu de la page se déroule naturellement, nous permettant de profiter du défilement natif au lieu d'écrire une solution de contournement de faux défilement. Les divs de l'entête et du pied de page sont donc dans le flux du document, mais chaque fois qu'ils sont hors de l'affichage, le framework les re-positionnera dynamiquement dans la vue si le navigateur supporte cette fonctionnalité, sinon ils vont tout simplement rester en ligne.

Pour activer ce comportement sur ​​une entête ou un pied de page, ajoutez l'attribut data-position="fixed" au conteneur de la barre d'outils.

Démonstration de formulaires dans des barres d'outils fixes

Appuyez pour choisir la visibilité

Pour choisir la visibilité des barres d'outils fixes, appuyez sur l'écran. Par exemple, si les barres d'outils fixes sont visibles, appuyez sur l'écran pour cacher les barres d'outils et pour profiter pleinement de l'écran complet pour le contenu. En tapant de nouveau cela ramènera l'affichage des barres d'outils.

Il est possible de fermer le robinet du comportement de bascule de la visibilité comme ceci :


$.mobile.fixedToolbars
   .setTouchToggleEnabled(false);

Mise à jour du positionnement de la barre d'outils

Si la hauteur de la page change, soit par l'injection dynamique de balisage, ou par des widgets qui cachent ou plient le contenu, cela peut rejeter le positionnement dynamique des barres d'outils. Pour demander manuellement à la barre d'outils de se repositionner, puis apparaitre en fondu, utilisez $.mobile.fixedToolbars.show();. Pour les faire apparaître immédiatement, sans le fondu :


$.mobile.fixedToolbars
   .show(true);

Il y a aussi un événement updatelayout qui peut être utilisé pour déclencher les barres d'outils pour se repositionner. Les développeurs qui créent des applications dynamiques qui injectent du contenu dans la page en cours peuvent également déclencher manuellement cet événement updatelayout pour assurer la mise à jour des composants sur la page en réponse à du nouveau contenu qui vient d'être ajouté. Cet événement est utilisé en interne dans les plugins de pliable et de filtre de liste et il est puissant parce qu'il n'est pas spécifique à la barre d'outils, n'importe quel widget peut être construit pour écouter l'événement updatelayout pour mettre à jour le widget dans la réponse.

Limitations connues

jQuery Mobile utilise dynamiquement les barres d'outils repositionnées pour l'effet d'entête fixe, car très peu de navigateurs mobiles supporte la propriété CSS position:fixed. Bien que notre fonctionnalité de barre d'outils fixe fonctionne assez bien, il y a un certain nombre de limitations techniques qui peuvent faire que les barres d'outils semblent défiler avec la page. La plupart de ces problèmes de rendu sont dus au fait que de nombreuses plateformes mobiles (iOS, Android, etc) essentiellement prennent une capture d'écran statique de la page et affichent cette image pendant le défilement au lieu d'utiliser le rendu HTML réel. Ceci améliore la performance du défilement, mais quand le défilement s'effectue rapidement, les barres d'outils seront "brûlées" dans la copie d'écran de page avant que notre script ne puisse les cacher ainsi elles semblent défiler avec la page. Nous avons optimisé cela autant que nous le pouvions, mais il va y avoir des situations où les barres d'outils fixes ne fonctionneront pas parfaitement à cause des limitations du navigateur, donc il est important de le souligner si vous souhaitez utiliser cette fonctionnalité.

Des véritables barres d'outils fixes : touchOverflowEnabled

Afin de parvenir à de véritables barres d'outils fixes, un navigateur a besoin de supporter position:fixed ou overflow:auto. Heureusement, ce support arrive sur les plateformes mobiles afin que nous puissions y parvenir avec les standards du web. Dans jQuery Mobile, nous avons ajouté une fonction globale appelée touchOverflowEnabledqui exploite la propriété CSS overflow:autosur les plateformes supportées, comme iOS5. Lorsqu'elle est activée, le framework encapsule chaque page dans un conteneur avec son défilement interne. Cela nous permet de positionner les barres d'outils en dehors du corps de défilement afin qu'elles restent véritablement en place tout le temps. Pour en savoir plus sur cette fonctionnalité sur les options globales de page ou la démonstration de cette fonctionnalité (actuellement iOS5 seulement, les autres navigateurs se rabattront dynamiquement sur les barres d'outils fixes repositionnés).
