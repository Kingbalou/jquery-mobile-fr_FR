Dans les navigateurs qui supportent position: fixed, (généralement, la plupart des navigateurs de bureaux, iOS5+, Android 2.2+, BlackBerry 6, et d'autres encore), les barres d'outils qui utilisent le plugin "fixedtoolbar" seront fixées à la partie supérieure ou inférieure de la fenêtre, tandis que le contenu de la page défilera librement entre les deux. Dans les navigateurs qui ne supportent pas le positionnement fixe, les barres d'outils se replieront vers du statique, le positionnement est intégré dans la page.

Pour activer ce comportement sur ​​une entête ou un pied de page, ajoutez l'attribut data-position="fixed" à un élément header ou footer de JQuery Mobile.

Barre d'outils avec entête fixe

	
<div data-role="header" data-position="fixed">
	<h1>Entête Fixe !</h1>
</div>

Barre d'outils avec pied de page fixe

	
<div data-role="footer" data-position="fixed">
	<h1>Pied de page fixe !</h1>
</div>

Barres d'outils pleines écrans

Les barres d'outils "plein écran" se positionnent en haut du contenu à chaque fois qu'elles sont visibles, et à la différence des barres d'outils fixes habituelles, les barres d'outils "plein écran" ne se replient pas vers un positionnement statique lors d'un basculement (afficher/masquer), au lieu de cela elles disparaissent entièrement de l'écran. Les barres d'outils "plein écran" sont idéales pour les interfaces plus immersive, comme une visionneuse de photos qui est censé remplir tout l'écran avec la photo. Cette page montre les barres d'outils qui utilise l'option fullscreen.

Pour activer cette option sur une entête ou un pied de page fixe, ajouter l'attribut data-fullscreen à l'élément.

	
<div data-role="header" data-position="fixed" data-fullscreen="true">
	<h1>Entête fixe !</h1>
</div>

Les changements dans jQuery Mobile 1.1

Avant la version 1.1, jQuery Mobile utilisait dynamiquement les barres d'outils repositionnées pour donner l'effet d'une entête fixe, car très peu de navigateurs mobiles supportaient la propriété CSS position:fixed. Et en simulant une fixation grâce à l'utilisation de "faux" JavaScript, le comportement du débordement du défilement réduisait le nombre de navigateur supporté, en plus cela se comportait de façon anormale sur certaines plateformes. Ce comportement n'était pas idéal et jQuery Mobile 1.1 a pris une nouvelle approche pour les barres d'outils fixes qui permet un support beaucoup plus large. Maintenant le framework offre de véritables barres d'outils fixes sur les nombreuses plateformes populaires, au lieu d'un positionnement statique dégradé pour les plateformes ne pouvant le supporter.

N'est plus pris en charge : touchOverflowEnabled

Avant la version 1.1 de jQuery Mobile, le véritable support de la barre d'outils fixe dépendait du support natif du navigateur avec la propriété CSS overflow-scrolling: touch, qui est actuellement pris en charge que par iOS5. Depuis la version 1.1, jQuery Mobile n'utilise plus du tout cette propriété CSS. Nous avons enlevé tous les usages internes de cette propriété dans le framework, toutefois, nous avons laissé l'objet $.mobile défini globalement pour éviter que sa suppression puisse causer des problèmes avec les applications existantes. Cette propriété est marquée pour suppression, donc s'il vous plaît mettez à jour votre code pour ne plus l'utiliser. Cependant, le test du support pour cette propriété reste défini sous $.support et nous n'avons pas l'intention d'enlever ce test pour l'instant.
