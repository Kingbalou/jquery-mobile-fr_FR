Le plugin popup a les options suivantes :

overlayTheme string

par défaut : null

Définit le jeu de couleurs (nuancier) pour le fond du popup, qui recouvre entièrement la fenêtre. S'il n'est pas explicitement définie, le fond sera transparent.

$('.selector').popup({ overlayTheme: "a" });

Cette option est également définie comme un attribut de données : data-overlay-theme="a"

theme string

par défaut : null

Définit le jeu de couleurs (nuancier) pour le contenu du popup. Sauf si il est explicitement défini à 'none', le thème pour le popup sera défini la première fois par l'héritage du thème de la page ou en cas d'échec par la valeur 'c' codé en dur. Si vous le définissez à 'none', le popup n'aura pas de thème du tout, et sera transparente.

$('.selector').popup({ theme: "a" });

Cette option est également définie comme un attribut de données : data-theme="a"

initSelector CSS selector string

default: ":jqmData(role='popup')"

Ceci est utilisé pour définir les sélecteurs (types d'éléments, les rôles de données, etc) qui seront automatiquement initialisé comme des popups. Pour changer les éléments qui sont initialisés, liez cette option à l'événement mobileinit :

$( document ).bind( "mobileinit", function(){
   $.mobile.popup.prototype.options.initSelector = ".mypopup";
});
shadow boolean

Par défaut : true

Définit s'il faut dessiner une ombre autour du popup. Cette option est également définie comme un attribut de données : data-shadow="true"

$('.selector').popup({ shadow: true });
corners boolean

Par défaut : true

Définit s'il faut dessiner un popup avec des coins arrondis. Cette option est également définie comme un attribut de données : data-corners="true"

$('.selector').popup({ corners: true });
