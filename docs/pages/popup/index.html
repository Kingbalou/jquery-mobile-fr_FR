Pour créer un popup, ajoutez l'attribut data-role="popup" au div avec le contenu du popup. Ensuite créez un lien avec href défini avec l'id du div du popup et ajoutez l'attribut data-rel="popup" pour appeler le framework afin d'ouvrir le popup lorsqu'il sera touché. Il s'agit d'un modèle de balisage similaire au widget de la boîte de dialogue.


<a href="#popupBasic" data-rel="popup">Open Popup</a>

<div data-role="popup" id="popupBasic">
	<p>This is a completely basic popup, no options set.<p>
</div>

Cela se traduira par le popup suivant :

Ouvrir le Popup

Il s'agit d'un popup de base, sans options définies.

Le framework ajoute de petite marge pour des éléments de texte, mais, c'est juste un conteneur avec des coins arrondis et une ombre qui sert d'ardoise vierge pour vos conceptions (même ces fonctions peuvent être désactivées via options) . Ce style simple permet d'ajouter facilement des widgets comme une listview pour faire un menu ou une photo pour faire une lightbox qui va jusqu'au bord du popup.

Appel du plugin popup

Ce plugin sera auto-initialiser sur n'importe quelle page qui contient un div avec l'attribut data-role="popup". Cependant, si nécessaire, vous pouvez appeler directement le plugin popup sur n'importe quel sélecteur, comme n'importe quel plugin jQuery et travailler par programmation avec l'API du popup via les options, les méthodes et les événements :


$('#myPopupDiv').popup();

Ouverture des popups

En utilisant la configuration basée sur le balisage, lorsqu'un lien avec data-rel="popup" est touché, le conteneur du popup correspondant à l'id référencé dans le href du lien sera affiché. Pour ouvrir un popup par programmation, appelez le popup avec la méthode open sur le conteneur du popup :


$('#myPopupDiv').popup("open")

Fermeture des popups

Les popups peuvent être fermés soit en cliquant en dehors du widget popup, soit en pressant la touche Esc. Les popups peuvent aussi être fermés via la méthode close : 


$('#myPopupDiv').popup("close")

Pour ajouter un bouton fermer explicite au popup, ajoutez un lien avec le rôle button dans le conteneur du popup avec un attribut data-rel="back" qui fermera le popup lorsqu'il sera touché. Nous avons créer des classes d'aide pour positionner les boutons dans le coin supérieur gauche (ui-btn-left) ou le coin supérieur droit (ui-btn-right) du popup, mais vous devrez peut-être modifier ou ajouter ces styles de positionnement selon votre conception.


<div data-role="popup">
	<a href="#" data-rel="back" data-role="button" data-theme="a" data-icon="delete" data-iconpos="notext" class="ui-btn-right">Fermer</a>
	...le contenu du popup se place ici...
</div>

Ajout d'espacement

Pour les popups avec du texte, l'espacement est nécessaire. Nous recommandons l'ajout de la classe ui-content au conteneur du popup qui ajoutera l'espacement standard de 15px, comme le conteneur contenant une page. Si vous avez besoin, écrivez vos propres styles pour créer un design plus personnalisé.


<a href="#popupPadded" data-rel="popup" data-role="button">Popup avec espacement</a>

<div data-role="popup" id="popupPadded" class="ui-content">
	<p>C'est un popup avec la classe <code>ui-content</code> ajouté au conteneur du popup.</p>
</div>

Cela se traduira par le popup avec de l'espacement suivant :

Popup avec espacement

C'est un popup avec la classe ui-content ajouté au conteneur du popup.

Lorsque l'espacement est ajouté, nous appliquons quelques règles de style pour annuler la marge supérieure de l'entête ou d'un paragraphe dans le popup et faisons de même pour la marge du bas du dernier élément. Cela permet de éviter un espace vertical trop important les popups, lorsque l'espacement du contenu et les marges des éléments se combinent.

Paramétrage des transitions

Par défaut, les popups s'ouvrent avec la transition pop. Ce "par défaut" est contrôlé via l'option globale $.mobile.defaultDialogTransition. Pour définir la transition utilisée par le popup, ajoutez l'attribut data-transition au lien qui référence le popup. 


<a href="#transitionExample" data-transition="none" data-rel="popup">
   Pas de transition
</a>

Pour des raisons de performances sur les appareils mobiles, nous vous recommandons d'utiliser des transitions simples comme pop, fade ou non pour avoir des animations de popup fluide et rapide, en particulier avec de larges ou de complexes widgets dans un popup.

Pas de transition Pop Fade Flip Turn Flow Slide Slidefade Slide up Slide down

Je suis une simple popup.

Lorsque vous lancez le popup depuis l'un des boutons, la data-transition donnée pour ce bouton sera utilisée. Toutefois, si vous lancez le popup par d'autres moyens, tels que, par exemple, via $("#transitionExample").popup("open"), la data-transition donnée pour le popup sera utilisée (la transition flip).

Thématisation du popup et de son fond

Le plugin popup fournit 2 options de thème : data-theme et data-overlay-theme. L'option data-theme fait référence au thème du popup, alors que data-overlay-theme se réfère au thème du fond du popup, qui couvre toute la fenêtre derrière le popup.

data-theme héritera de la page et aura toujours une valeur valide lorsque le popup s'ouvrira, à moins que vous spécifiez explicitement data-theme="none", dans ce cas, le popup aura un fond transparent.

Le data-overlay-theme ne sera jamais défini et le popup de fond, bien qu'il soit toujours présent lorsque le popup s'affiche, sera complètement transparent, à moins explicitement de le définir en utilisant par exemple data-overlay-theme="a". Dans ce cas, l'arrière-plan sera fade in, cachant partiellement le reste de la fenêtre, pour attirer l'attention sur le popup Voici un exemple d'un popup thématisé :


<div id="both" data-role="popup" data-theme="e" data-overlay-theme="a" class="ui-content">
  ...Contenus du popup...
</div>

Remarque : Le chainage des popups n'est pas permis

Le framework ne prend pas en charge le chaînage des popups car il n'est pas possible d'intégrer un lien d'un popup à l'intérieur d'un autre popup. Tous les liens avec data-rel="popup" à l'intérieur d'un popup ne fera rien du tout.
